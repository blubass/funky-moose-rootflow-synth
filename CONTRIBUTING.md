# Contributing to RootFlow

Thank you for your interest in contributing to RootFlow! We welcome all kinds of contributions, from bug reports to new features and UI improvements.

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue on GitHub. Include as much detail as possible:
- Your OS version and DAW.
- Steps to reproduce the issue.
- Expected vs. actual behavior.
- Screenshots or audio clips if applicable.

### Suggesting Features
Have an idea to make RootFlow even more organic? Open an issue with the "feature request" tag and describe your vision.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Follow the existing coding style (JUCE-based C++).
4. Ensure your code builds on both macOS and Windows.
5. Open a Pull Request with a clear description of your changes.

## Development Setup

See [BUILDING.md](docs/BUILDING.md) for detailed build instructions.

## Code of Conduct

Please be respectful and constructive in all interactions within this project.
